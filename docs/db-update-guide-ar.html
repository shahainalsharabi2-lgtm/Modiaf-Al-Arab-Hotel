شرح عملية «تحديث قاعدة البيانات» — فرونت-اند + باك-اند

*مشروع مضياف العرب للفنادق v2.3.3*

1. نظرة عامة

عند الضغط على «تحديث قاعدة البيانات» في لوحة إعدادات DB، تمر العملية عبر:

1. الواجهة (Angular) — زر → خدمة HTTP → طلب PUT

2. الشبكة — proxy محلي أو Cloudflare Worker

3. الخادم (ASP.NET + ABP) — App Service → Migrations → Seed

4. قاعدة البيانات — PostgreSQL (إنتاج) أو SQL Server (محلي)

العملية = خطوتان على الخادم:

  • Migrations — تحديث هيكل الجداول والأعمدة
  • Seed — إكمال البيانات الافتراضية الناقصة

8. التسلسل الزمني (خطوة بخطوة)


1. المستخدم يضغط زر الحساب → toggleDbPanel()
2. تفتح لوحة db-settings-panel
3. يضغط «تحديث قاعدة البيانات»
4. updateDatabase() → updateLoading = true
5. PUT /api/app/hotel-database-admin/database
6. proxy (محلي) أو worker.ts (إنتاج) → Render API
7. HotelDatabaseAdminAppService.UpdateDatabaseAsync()
8. EntityFrameworkCoreHotelDbSchemaMigrator.MigrateAsync()
9. EF Core يطبّق Migrations على PostgreSQL
10. IDataSeeder → كل DataSeedContributor
11. JSON { success, message } → Angular
12. عرض الرسالة في db-panel__status

9. ماذا يحدث لبياناتك؟

العمليةالتأثير
Migrationsيضيف/يعدّل أعمدة وجداول — لا يحذف الغرف أو الحجوزات
Seedيكمّل الناقص فقط
لا يوجدDROP TABLE، مسح حجوزات، استبدال كامل

نصيحة: قبل التحديث على الإنتاج → «إنشاء نسخة احتياطية» (POST /backup).
