# خريطة المجالات بين UltimateOtelPremio ومضياف العرب

*مرجع الهيكلية: UltimateOtelPremio (ABP Modular Monolith) — المشروع الحالي: Modiaf.Al.Arab.Hotel*

---

## 1. مقارنة البنية العامة

| الجانب | UltimateOtelPremio | Modiaf.Al.Arab.Hotel |
|--------|-------------------|----------------------|
| النمط | Modular Monolith | انتقال تدريجي → Modular |
| الباك-اند | `modules/` + `host/` | `aspnet-core/modules/` + `aspnet-core/src/` |
| الواجهة | Angular (مكتبات لكل module) | Angular (`angular/src/app/` مجلدات feature) |
| DbContext | واحد لكل وحدة + دمج في Host | `HotelDbContext` يدمج وحدات عبر `ReplaceDbContext` |
| المصادقة | AuthServer منفصل | OpenIddict داخل HttpApi.Host |

## 3. وحدة Reservations (المنفذة)

### 3.1 هيكل المشاريع

```
aspnet-core/modules/Reservations/
├── Modiaf.Al.Arab.Hotel.Reservations.Domain.Shared/
├── Modiaf.Al.Arab.Hotel.Reservations.Domain/
├── Modiaf.Al.Arab.Hotel.Reservations.Application.Contracts/
├── Modiaf.Al.Arab.Hotel.Reservations.Application/
├── Modiaf.Al.Arab.Hotel.Reservations.EntityFrameworkCore/
└── Modiaf.Al.Arab.Hotel.Reservations.HttpApi/
```

### 3.2 المحتوى الحالي

| الطبقة | الملفات الرئيسية |
|--------|------------------|
| Domain | `Booking` entity, `IBookingRepository` |
| Application.Contracts | `BookingDto`, `CreateUpdateBookingDto`, `IBookingAppService` |
| Application | `BookingAppService` |
| EntityFrameworkCore | `IReservationsDbContext`, `EfCoreBookingRepository`, `BookingEntityTypeConfiguration` |
| HttpApi | `ReservationsHttpApiModule` |

### 3.3 الدمج مع المضيف

- `HotelDbContext` ينفّذ `IReservationsDbContext` مع `[ReplaceDbContext]`
- جدول قاعدة البيانات لم يتغير: `AppBookings`
- مسار API لم يتغير: `/api/app/booking`
- namespace الكيانات: `Modiaf.Al.Arab.Hotel.Bookings` (توافق مع Angular والـ migrations)

### 3.4 تبعيات الوحدات في المضيف

```
HotelHttpApiHostModule
  → HotelHttpApiModule (+ ReservationsHttpApiModule)
  → HotelApplicationModule (+ ReservationsApplicationModule)
  → HotelEntityFrameworkCoreModule (+ ReservationsEntityFrameworkCoreModule)
```

## 5. خطة الانتقال التدريجي

| الأولوية | الوحدة المقترحة | الكيانات المرشحة |
|----------|-----------------|------------------|
| 1 | ✅ Reservations | Booking |
| 2 | RoomsManagement | Room, RoomType, Floor |
| 3 | FrontDesk | GuestRegistry, Keys (مستقبلاً) |
| 4 | HouseKeeping | مهام HK (عند ربط API) |
| 5 | NightAudit | إجراءات التدقيق الليلي |
| 6 | ProfileManagement | CRM entities |
| 7 | RevenueManagement | Cashier, Accounts |

### خطوات إنشاء وحدة جديدة (نفس نمط Reservations)

1. إنشاء 6 مشاريع تحت `aspnet-core/modules/{Name}/`
2. نقل Entity + Repository + AppService + DTOs
3. تعريف `I{Name}DbContext` + `Configure{Name}()` في EF
4. دمج في `HotelDbContext` عبر `ReplaceDbContext`
5. إضافة `DependsOn` في وحدات المضيف
6. تسجيل `ConventionalControllers` للـ assembly الجديد
7. إضافة المشاريع إلى `Modiaf.Al.Arab.Hotel.slnx`
